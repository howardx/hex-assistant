# Communication Style

How all output should be written for hex2.

- **Format:** Bullet points with supporting detail. Not verbose — tight and to the point.
- **Tone:** Casual, for both internal and external/public-facing content.
- **No emojis.** Ever.
- **No marketing fluff** — no hype, filler, or upsell language. Plain and direct.
- Lead with the answer; put detail after.
